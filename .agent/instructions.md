# Workspace Instructions

## Testing and Verification Policy
* Unless explicitly instructed otherwise, **do not run tests or automated verifications** (such as adb taps/typing, screenshots, logcat checks, or browser subagents).
* Ensure the code fix is correctly implemented.
* Verify that the application builds and deploys successfully.
* Ensure the changes can be build and run via android studio.
* The user will perform the actual tests and verification.
* **Do not automatically commit changes** to the repository unless explicitly instructed by the user.
